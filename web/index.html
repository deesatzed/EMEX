EMEX

synthetic manual OE clinician draft
Workflow demo only. The CLI/backend artifact path is authoritative for safety checks and audit artifacts.

Provider Workflow

1

Load clinical information.
Paste or upload the deidentified/synthetic case context gathered from the EHR outside EMEX.

2

Prepare OE input.
Click Prepare OE. EMEX redacts identifiers and blocks post-triage leakage before showing copy-ready text.

3

Run DotFlows in OpenEvidence.
Copy the redacted packet into OE. Use this sequence:

First: ed_shadow_recommenderPrimary ED shadow assessment for every case.
Second: order_appropriateness_reviewChecks proposed evaluation considerations for missing, duplicative, excessive, or low-value elements.
Third: risk_resource_forecastForecasts draft risk/resource intensity without disposition or bed authority.
Fourth: cost_restraint_reviewReviews cost, duplication, and low-value testing cautions after safety is addressed.
Last: discordance_reviewUse only for after-the-fact PI review once actual provider actions/outcomes are available.
4

Paste OE output back into EMEX.
Paste the OE result into OE Output, then click Parse OE to create structured clinician-draft suggestions.
